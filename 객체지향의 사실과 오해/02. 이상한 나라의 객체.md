# 02. 이상한 나라의 객체

## **1. 객체지향과 인간의 인지 능력**

- 우리가 세상을 작은 단위(객체)로 나누어 이해하는 이유는 **복잡성을 극복하기 위해서**이다.
- 객체는 **사람이 구별할 수 있는 경계**를 가진 물리적/개념적 존재이다.
- 소프트웨어 속의 객체 세계는 현실과 닮았지만 실제로는 매우 **이질적인 규칙**을 갖는다.

---

## **2. 이상한 나라의 앨리스와 객체**

### **앨리스 객체가 알려주는 객체의 본질**

- 앨리스의 **행동이 상태를 바꾼다.**
    
    → 먹으면 키가 커지고, 마시면 작아지는 것처럼.
    
- 그러나 행동의 결과는 **현재 앨리스의 상태에 따라 달라진다.**
- 상태가 변해도 **식별자(정체성)는 바뀌지 않는다.**
    
    → 키가 작아져도 앨리스는 여전히 앨리스.
    

➡ 객체는 **상태·행동·식별자**라는 3요소로 설명해야 가장 명확하다.

---

## **3. 소프트웨어 속 객체**

### **객체란?**

- **식별 가능한 개체** 또는 **추상적 개념**.
- **변하는 상태**, **특징적 행동**, **식별자**를 가진다.
- 소프트웨어에서는
    
    → **상태 = 저장된 값들(필드)**
    
    → **행동 = 실행 가능한 코드(메서드)**
    
    로 구현된다.
    

---

# **4. 상태(State)**

### **왜 상태가 필요한가?**

- 객체의 행동 결과는 **현재까지 무엇이 일어났는지**에 의해 결정된다.
- 하지만 과거 행동 전체를 기억하는 것은 불가능하므로, 인간은 이를 **“상태”라는 개념으로 요약**한다.
- 상태가 있기 때문에 **행동의 결과를 쉽게 예측**할 수 있다.

### **상태 = 프로퍼티(속성 + 링크)**

- **단순 값(숫자, 문자열)** → 객체의 속성(attribute) 역할
- **다른 객체와의 연결(link)** → 객체 간 관계를 표현
- 결국 객체의 상태는
    
    → **속성 + 링크**로 구성된다.
    

---

# **5. 행동(Behavior)**

### **행동의 핵심 특징**

1. **객체의 상태를 바꿀 수 있는 것은 객체 자신의 행동뿐이다.**
    
    → 외부가 직접 상태를 변경할 수 없다.
    
2. **행동의 결과는 현재 상태에 의존한다.**
    
    → 문을 통과할 때 키가 크냐 작으냐에 따라 결과가 달라지는 것처럼.
    
3. **행동은 상태를 변경한다(부수효과).**
4. 객체는 **상태를 숨기고 행동만 외부에 노출**한다.
    
    → 캡슐화.
    
5. 상태를 감추고 행동을 강화할수록
    
    → **객체는 더 자율적이고 지능적으로 보인다.**
    
    → 협력은 더 단순하고 유연해진다.
